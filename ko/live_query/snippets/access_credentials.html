액세스 자격 증명 얻기
이 SDK가 지원하는 가장 효과적인 기능은 플레이어 데이터를 쿼리하거나 Cryptoitems에서 작업을 요청하려면 액세스 토큰이 필요하다는 것입니다.

이 방식은 사용자의 자격을 증명할 수 있는 액세스 토큰을 검색하는 것입니다. 이 액세스 토큰은 엔진의 클라우드 플랫폼에 대한 아웃 바운드 POST 요청의 ‘Authorization’ 헤더에 포함되어 사용됩니다. 액세스 토큰은 사용자를 식별하고 유효한 사용 권한을 가진 사용자만 클라우드 플랫폼 작업을 시행할 수 있도록 합니다. 이 예시를 시행하면 이 페이지에 나온 다른 예시들을 시행하는데 필요한 엑세스 토큰을 설정합니다.

클라우드 플랫폼에 로그인 하기 위해서는 이메일 주소와 암호가 필요합니다. 이를 실행하기 위한 GraphQL 쿼리는 다음과 같습니다: 

query login($email:String!, $password:String!) {
  results: EnjinOauth(email: $email, password: $password) {
    id
    access_tokens
    roles {
      name
    }
    identities {
      id
      app_id
      ethereum_address
    }
  }
}

‘Submit Query’ 버튼을 누르면 위에 있는 쿼리가 아래에 추가된 파라미터와 함께 클라우드 플랫폼 엔드 포인트에 POST됩니다: 


{}

다음은 클라우드 플랫폼에 출력된 응답입니다: 

{}

Unity SDK에서는 Enjin.Login(string username, string password) 과 같은 방식으로 구현됩니다. 이 방법은 사용자 ID, 액세스 토큰, 역할 배열 및 ID 배열이 포함된 사용자 객체를 반환하기 전에 같은 사용자 이름과 암호를 입력해야 합니다. 비슷한 맥락에서, Enjin.VerifyLogin(string username, string password) 방식은 동일한 로그인 쿼리를 수행하지만, 사용자, 성공 상태만 반환하며 글로벌 AppID 상태는 변경하지 않습니다. 이 목적은 요청을 대신하는 게임 서버에 대한 사용자의 올바른 로그인을 확인하기 위해서 입니다. 또한, SDK는 StartPlatform() 방식이 포함되어 있어, SDK 전체를 특정 클라우드 플랫폼에 초기화하여 특정 사용자가 요청을 발행할 수 있게 합니다. 이 방식은 관리자가 로그인 할 때 유용하게 사용됩니다.
