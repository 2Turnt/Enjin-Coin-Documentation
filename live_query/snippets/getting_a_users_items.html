Getting a User's Items
With access token in hand, we can retrieve a User's Items.

This method, given a User's access token, will send a request to the Cloud Platform which returns a User's inventory for a specified Identity. An Identity is tied to a single wallet address, and a User can have many of them across different Cloud Platform apps. Users can also have a single Identity with items from multiple apps.

Retrieving a User's inventory from the Cloud Platform requires knowing the ID for an Identity of theirs. The GraphQL query to do so is as follows: 


query getAllItems($identityId: Int!) {
  result: EnjinIdentities(id: $identityId) {
    tokens(include_creator_tokens: true) {
		app_id
		index
		token_id
		name
		creator
		totalSupply
		reserve
		circulatingSupply
		supplyModel
		meltValue
		meltFeeRatio
		meltFeeMaxRatio
		transferable
		transferFeeSettings {
		  type
		  token_id
		  value
		}
		nonFungible
		icon
		isCreator
	}
  }
}

Pressing the 'Submit Query' button will POST the query above to the Cloud Platform endpoint, along with additional parameters encoded as follows: 


{}

Here is the output response from the Cloud Platform: 


{
}

In this example query, we have retrieved the first page of a User's inventory from the Cloud Platform. Please note that the "page" parameter may be changed to query for additional items as desired by your game's needs. The corresponding function in the Unity SDK to this GraphQL call is Enjin.GetAllItems(int identityID) which will return a workable list of all Items tied to an Identity and their relevant metadata.
